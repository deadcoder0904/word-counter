# Repository Guidelines

## Project Structure & Module Organization
**Core paths**
- `src/App.tsx` holds the main UI; `src/main.tsx` boots the React app.
- `src/hooks/` contains reusable hooks (example: `useTextStats.ts`).
- `src/utils/` contains shared logic (example: `textStats.ts`).
- `src/index.css` defines global styles; Tailwind is wired via Vite.
- `dist/` is build output (generated by `vite build`).

**Entry points**
- `index.html` is the Vite HTML entry.
- No dedicated assets directory yet; add `src/assets/` if needed.

## Build, Test, and Development Commands
**Local workflow**
- `bun install` installs dependencies (or `npm install`).
- `bun run dev` starts the Vite dev server.
- `bun run build` outputs production assets to `dist/`.
- `bun run preview` serves the built app locally.

**Quality checks**
- `bun run lint` runs oxlint on the repo.
- `bun run typecheck` runs oxlint in type-aware mode.
- `bun run format` formats code with oxfmt.
- `bun run format:check` verifies formatting without writing.
- `bun run clean` removes `dist/`.

Quick dev loop:
```bash
bun install
bun run dev
bun run lint
```

## Coding Style & Naming Conventions
**Formatting (oxfmt)**
- 2-space indentation, single quotes, semicolons, 100-char line width.
- Trailing commas in ES5 positions (arrays/objects).

**Naming**
- Components: `PascalCase.tsx` in `src/`.
- Hooks: `useSomething.ts` in `src/hooks/`.
- Utilities: `camelCase` exports in `src/utils/`.

## Testing Guidelines
**Current state**
- No test framework is configured; add tests alongside new behavior.
- If introducing tests, document the chosen location (`tests/` or `src/__tests__/`).

## Commit & Pull Request Guidelines
**Commits**
- This folder does not include Git metadata, so no conventions are visible.
- Use short, imperative subjects (e.g., `ui: refine word count header`).

**Pull requests**
- Include a short summary, testing steps, and screenshots for UI changes.
- Note any user-visible behavior changes and update docs if needed.

## Security & Configuration Tips
**Secrets**
- No `.env` patterns exist yet; add `.env` for API keys and avoid commits.
- Document new configuration in the repository root docs when introduced.
